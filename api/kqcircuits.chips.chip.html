kqcircuits.chips.chip¶

class kqcircuits.chips.chip.Chip[source]¶

Bases: kqcircuits.elements.element.Element

Base PCell declaration for chips.

By default produces in face 0 the chip frame consisting of texts in pixel corners, dicing edge, markers and optionally grid. Production of the chip frames can be adjusted by overriding the produce_frames() method.

Provides helpers to produce launchers and junction tests.

produce_n_launchers(n, launcher_type, launcher_width, launcher_indent, launcher_assignments, pad_pitch)[source]¶

Produces n launchers at default locations.

Parameters

  • n – number of launcher pads

  • launcher_type – type of the launchers, “RF” or “DC”

  • launcher_width – width of the launchers

  • launcher_indent – distance between the chip edge and pad port

  • launcher_assignments – dictionary of (port_id: name) that assigns a role to some of the launchers

  • pad_pitch – distance between pad centers

Returns

launchers as a dictionary {name: (point, heading, distance from chip edge)}

PCell parameters:

  • box (Shape) - Border, default=(0,0;10000,10000)

  • with_grid (Boolean) - Make ground plane grid, default=False

  • dice_width (Double) - Dicing width, default=200, unit=[μm]

  • name_mask (String) - Name of the mask, default=M99

  • name_chip (String) - Name of the chip, default=CTest

  • name_copy (String) - Name of the copy, default=None

  • dice_grid_margin (Double) - Margin between dicing edge and ground grid, default=100
